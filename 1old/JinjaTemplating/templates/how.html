How Jinja Works

Templates

  • You write Jinja templates (e.g., in an .html file) that contain a mix of static text and special Jinja syntax.
  • The Jinja include directive is a tag used to insert the rendered content of one template file into another
  • Template Inheritance allows you to define a base template and have other templates inherit and extend its structure.

Delimiters

Special syntax, or "delimiters," are used to indicate Jinja cod

  • {{ expression }}: Evaluates an expression and prints its result, like a variable
  • {% statement %}: Executes statements, such as loops or conditional statements, to control the template's behavior but doesn't produce output itself.
  • {# comment #}: For comments within the template, which are not rendered in the final output.

Data

A Python application provides data (variables) to the template

Rendering

Jinja processes the template, filling in the variables and executing the logic, to produce the final, dynamic document.
